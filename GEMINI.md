pprof to csv tools

## Plan

- support analyzing pprof profile with github.com/google/pprof packages
- support dumping source line to time in csv file.

## Architecture Design

### Components

1. **Profile Loader**
   - Reads pprof profile data from file.
   - Only need to support profile data generated by go test -cpuprofile and -memprofile.
   - Handles authentication for remote profiles

2. **Profile Analyzer**
   - Uses github.com/google/pprof/profile package to parse profile data
   - Extracts function call stacks and timing information
   - Maps program counters to source code locations
   - Aggregates time spent per source line

3. **CSV Exporter**
   - Converts analyzed data to CSV format
   - Outputs columns: filename, line number, function name, cumulative time, flat time
   - Configurable output options (time units, filtering, sorting)

4. **CLI Interface**
   - Command-line flags for input/output configuration
   - Support for multiple input files
   - Verbose/debug modes

```
pprof2csv/

│   └── loader.go    // Implements profile loading interface
├── analyzer/        // Profile analysis functionality  
│   └── analyzer.go  // Implements profile analysis interface
├── exporter/        // Export functionality
│   └── csv.go       // CSV export implementation
└── internal/        // Internal utilities
    └── utils.go     // Shared utilities
```

### Dependencies

```go
// go.mod
require (
    github.com/google/pprof v0.0.0-20230718154909-66e3cb66a1f8

```

### Main Workflow

1. Parse command-line arguments
2. Load pprof profile using Profile Loader
3. Analyze profile with Profile Analyzer to extract source line timing
4. Export results to CSV using CSV Exporter
5. Output success/failure status

### Implementation Steps

1. Initialize Go module with required dependencies
2. Create basic CLI structure in main.go
3. Implement Profile Loader interface
4. Implement Profile Analyzer using github.com/google/pprof
5. Implement CSV Exporter
6. Integrate components and test with sample pprof data
7. Add error handling and validation
8. Write documentation and examples
